# مستندات واجهة برمجة تطبيقات الإعلانات (Ads API)

الرابط الأساسي (Base URL): `/api/v1`

هذا الملف يوضح بالتفصيل مسارات (Endpoints) الخاصة بعمليات الإعلانات (عروض البيع والشراء) في المنصة ، كإضافة إعلان، تعديله، حذفه، استعادته، وتجديده.

جميع المسارات أدناه تستقبل وترجع استجابة بصيغة `JSON`، ماعدا المسارات التي تتطلب رفع صور (`multipart/form-data`).

---

## 1. جلب قائمة الإعلانات (List Ads)
عرض قائمة بجميع الإعلانات المتاحة. يدعم الفلترة والبحث.

- **المسار:** `GET /ads`
- **الحماية:** مسار عام (Public)

### معاملات البحث والفلترة المتاحة (Query Parameters)
- `category_id`: (اختياري) معرف القسم لفلترة الإعلانات.
- `search`: (اختياري) نص للبحث في عنوان أو وصف الإعلان.
- (معاملات أخرى يتم دعمها من `AdRepository` كالفرز والسعر ونحو ذلك)

### الاستجابة الناجحة (Success Response - 200 OK)
```json
{
  "success": true,
  "message": "data_retrieved",
  "data": {
    "current_page": 1,
    "data": [
      {
        "id": 1,
        "title": "تويوتا كامري 2023",
        "price": "85000",
        // ... بقية تفاصيل الإعلان
      }
    ],
    // ... تفاصيل التصفح (Pagination)
  }
}
```

---

## 2. تفاصيل الإعلان (Show Ad)
عرض تفاصيل إعلان محدد. يقوم هذا المسار برفع عداد المشاهدات (`views_count`) تلقائياً.

- **المسار:** `GET /ads/{id}`
- **الحماية:** مسار عام (Public)

### الاستجابة الناجحة (Success Response - 200 OK)
ترجع تفاصيل الإعلان شاملة (الصور، صاحب الإعلان، القسم، والتعليقات).

```json
{
  "success": true,
  "message": "data_retrieved",
  "data": {
    "id": 1,
    "title": "تويوتا كامري 2023",
    "description": "نظيفة جدا استخدام شخصي...",
    "views_count": 150,
    "user": { ... },
    "category": { ... },
    "images": [ ... ],
    "comments": [ ... ]
  }
}
```

---

## 3. إضافة إعلان جديد (Create Ad)
إضافة إعلان جديد من قبل المستخدم.
> **ملاحظة:** إضافة إعلان تتطلب رفع صور، لذلك يجب إرسال الطلب بصيغة `multipart/form-data`.
> **قيود:** يحق للمستخدم العادي إضافة 3 إعلانات فقط في اليوم. (أصحاب الاشتراكات بلا حدود ويكون إعلانهم مميز `is_featured`).

- **المسار:** `POST /ads`
- **الحماية:** محمي (Protected) - يتطلب إرسال التوكن (Bearer Token).

### البيانات المطلوبة (Request Body - `multipart/form-data`)
| الحقل | الوصف | قواعد التحقق (Validation) |
| :--- | :--- | :--- |
| `title` | عنوان الإعلان | مطلوب (نص، حتى 255 حرف، لا يحتوي كلمات بذيئة) |
| `description` | تفاصيل الإعلان | مطلوب (نص، على الأقل 10 أحرف) |
| `price` | السعر | مطلوب (رقم، أقل شيء 100) |
| `category_id` | معرف القسم | مطلوب (موجود في جدول الأقسام) |
| `address_text`| العنوان النصي | مطلوب |
| `lat` | خط العرض (اختياري) | رقم (بين 12 و 19) |
| `lng` | خط الطول (اختياري) | رقم (بين 42 و 54) |
| `year`| سنة الصنع (للسيارات) | اختياري (سنة صحيحة) |
| `images[]` | صور الإعلان | مطلوب (مصفوفة ملفات صور، من 1 إلى 5 صور كحد أقصى، كل صورة لا تتجاوز 2 ميجا) |

### الاستجابة الناجحة (Success Response - 201 Created)
```json
{
  "success": true,
  "message": "ad_created",
  "data": { ... } // تفاصيل الإعلان المنشأ مع الصور
}
```

### استجابة الخطأ لتجاوز الحد المسموح (Error - 403 Forbidden)
```json
{
  "success": false,
  "message": "daily_limit_reached"
}
```

## 4. تعديل الإعلان (Update Ad)
تحديث بيانات إعلان خاص بالمستخدم. إذا كان الإعلان نشطاً وتم تعديله، فسيتم إشعار الإدارة لاحتمالية المراجعة.
> **ملاحظة:** يمكن إضافة صور جديدة هنا بصيغة `multipart/form-data` مع بقية البيانات.

- **المسار:** `PUT /ads/{id}` (أو إرسال `POST` مع `_method=PUT` كالمعتاد في لارافل عند رفع الملفات)
- **الحماية:** محمي (Protected) - يتطلب التوكن (يجب أن يكون المستخدم هو صاحب الإعلان).

### البيانات المطلوبة
نفس بيانات الإضافة (title, description, price, ...)، ويمكن إرسال `images[]` لإضافة صور إضافية جديدة.

### الاستجابة الناجحة (Success Response - 200 OK)
```json
{
  "success": true,
  "message": "ad_updated",
  "data": { ... }
}
```

---

## 5. حذف الإعلان (Soft Delete)
حذف الإعلان بشكل ناعم. تظل الصور محفوظة في السيرفر لاستعادته لاحقاً متى ما أراد المستخدم.

- **المسار:** `DELETE /ads/{id}`
- **الحماية:** محمي (Protected) - يتطلب التوكن.

### الاستجابة الناجحة (Success Response - 200 OK)
```json
{
  "success": true,
  "message": "ad_deleted",
  "data": null
}
```

---

## 6. استعادة إعلان محذوف (Restore Ad)
استعادة الإعلان إذا تم حذفه مسبقاً من قبل صاحبه وإعادته متاحاً للعموم.

- **المسار:** `POST /ads/{id}/restore`
- **الحماية:** محمي (Protected) - يتطلب التوكن.

### الاستجابة الناجحة (Success Response - 200 OK)
```json
{
  "success": true,
  "message": "ad_restored",
  "data": { ... }
}
```

---

## 7. تجديد الإعلان (Renew Ad)
تجديد مدة إعلان (تأليف تاريخ الانتهاء). يعيد الإعلان ليكون تفاعلياً وربما يرفعه في قائمة الإعلانات (بحسب منطق التجديد في `AdRepository`).

- **المسار:** `POST /ads/{id}/renew`
- **الحماية:** محمي (Protected) - يتطلب التوكن.

### الاستجابة الناجحة (Success Response - 200 OK)
```json
{
  "success": true,
  "message": "ad_renewed",
  "data": { ... }
}
```
